⠿ ◈ ABS GEO Mapping 1 ▼
PBI Report
Data Engineering | Gov
Qlik Migration
Python | Automation
Other | BAU
✕ Remove group
⠿
Extends membership geographic analysis from postcode level down to State, Postcode, Suburb, and Street Name by enriching member addresses with precise coordinates via G-NAF Core — Australia's national authoritative address database maintained by PSMA — then performing a spatial join against ABS boundary shapefiles (SA1–SA4). Unlike a conventional table lookup, this uses point-in-polygon matching via GeoPandas (sjoin) — each member's coordinate is tested against the geometric boundaries defined in the ABS shapefile to determine which statistical area it falls within. This approach is fundamentally spatial rather than textual, enabling accurate classification even where postcodes span multiple regions.

⠿ ◈ Snapshot Backup 1 ▼
PBI Report
Data Engineering | Gov
Qlik Migration
Python | Automation
Other | BAU
✕ Remove group
⠿
Addresses the proliferation of daily snapshot tables in SQL Server — where each tracked table accumulates ~30 snapshots per month, creating bloat and degrading query performance over time. This notebook consolidates a configurable date range of snapshots across multiple tables into single Parquet files per table, enabling safe deletion of the source SQL snapshots. Parquet is chosen for its columnar compression, efficient storage, and native compatibility with Power BI, Python, and cloud platforms — making it a more appropriate format for archival than row-based SQL tables. Snapshot date is preserved as a column, allowing point-in-time queries against the archived file without needing to restore anything to SQL Server.
